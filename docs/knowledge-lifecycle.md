# 知识生命周期系统 (规划阶段)

系统在未来的演进将从“怎么生成知识”跨越到“怎么管理知识”，从单向的「知识生成系统」升级为具备记忆、判断、淘汰和进化能力的「知识生命周期系统」。这是个人认知操作系统的核心。

## 一、核心三大机制

为实现完整的生命周期管理，下一阶段（v0.7+）将引入以下三个底层机制与能力：

### 1. 知识使用追踪（Knowledge Usage）
**新增 Skill：`value-feedback-tracker`**

- **目标**：识别“哪些知识被使用”和“哪些没用”。
- **机制**：通过分析被搜索的频次、被其他知识引用的频次、以及被输出为文章/报告引用的频次，动态计算每个知识节点的权重。
- **输出**：生成定期的 `usage-report`，告知哪些长期沉淀没有产出任何价值。

### 2. 知识老化机制（Knowledge Aging）
**新增 Skill：`knowledge-aging-manager`**

- **状态流转**：引入时间衰减模型，自动将知识节点的状态沿以下生命周期流转：
  `active` → `stale`（长时间未访问） → `cold`（沉寂且未被验证） → `risk`（可能过时或产生冲突） → `archived`（封存）
- **自动触发**：当系统运行时，定期扫描 stale 和 cold 的文件，推入 manual_review，或自动加上 `archived` tag 以减小后续 LLM 的上下文负担。

### 3. 输出回流机制（Output Feedback Loop）
**新增 Skill：`output-linker`**

- **目标**：实现“文章/对外输出 → 回链知识库”。
- **机制**：当你根据本知识库完成了一篇正式文章后，将文章放入系统中，该机制会自动识别文章使用了哪些 source 和 concept，并反向在这些源笔记中打上“已成功应用”的标记，强化这些高价值节点的权重。

## 二、系统闭环

这三大机制将彻底打通知识在系统内的流动闭环：

**输入（raw/ingest） → 生长（seed/topic） → 使用（usage-tracker） → 回流（output-linker） → 进化/淘汰（aging-manager）**

## 三、优先级演进路线

1. **第一优先（v0.6）**：log + index 的结构化重构（目前正在进行中），建立清晰的基础导航体系以承载复杂的元数据管理。
2. **第二优先**：usage + aging 机制，解决“只有存”没有“丢”的熵增问题。
3. **第三优先**：output 回流，实现从“消费”到“产出”的认知闭环。

## 四、设计原则

1. **非破坏性**：不破坏现有知识结构，不任意删除用户文件，仅修改元数据与目录路径。
2. **可追溯性**：所有新机制（包括封存、降级）必须具备溯源能力。
3. **安全门禁**：高风险操作（大批量归档、删除无效链接）必须进入 `manual_review`。
4. **日志记录**：所有的生命周期状态变化，必须如实记录在结构化 log 中。
