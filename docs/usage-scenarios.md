# 个人知识库管家 (Personal KB Steward) - 全景使用场景演示

本指南旨在通过具体的场景与对应的命令/提示词示例，向你展示管家在不同阶段是如何调用各个 Skill 协同工作的。

管家系统遵循**“平时自动生长，战时按需征召”**的理念。系统包含的技能（Skill）可分为四大核心场景：

---

## 场景一：日常摄入与无感沉淀 (Daily Ingestion)

**典型情境**：你白天在微信、网页端或各种信息流中遇到了有意思的文章片段、参加了一场重要会议、或者保存了一份 30 页的行业研报。你将它们全部粗暴地丢进知识库的 `raw/` 或 `inbox/` 文件夹中。

**执行命令**（通常设为每日定时任务或手动执行）：
```bash
python scripts\personal_kb_steward.py run --apply
```

**背后调用的技能与效果**：

1. **`raw-ingest-router` (长文路由器)**
   - **它的工作**：扫描 `raw/` 中未经处理的文件，让 LLM 判断内容属性。
   - **系统级 Prompt 示例**：“请判断这篇文章属于：灵感短记(seed)、会议工作记录(work_memory)、还是深度调研长文(research_report)？”
   - **结果**：将你的凌乱素材分发给对应的专员。

2. **`work-memory-weave` (工作记忆编织)**
   - **它的工作**：接管上面分发来的会议纪要，提取决定和待办。
   - **内部 Prompt 示例**：“请从这段杂乱的会议速记中，提取出：1. 已确认的关键决策 2. 各人的待办事项。”

3. **`mindseed-grow` (灵感种子萌发)**
   - **它的工作**：接管带有 `#seed` 标签的零碎想法和短文，寻找它们之间的共性，把 5 个散落的碎片聚合成一个 `seed-cluster` 节点。
   - **内部 Prompt 示例**：“以下有 N 个零碎笔记，请判断哪些讨论了相同的核心概念，将它们聚类并提炼出一个有意义的标题（如：AI 认知摩擦）。”

4. **`topic-research-compile` (长文主题提炼)**
   - **它的工作**：接管 30 页的行业研报。不只是做总结，而是提取“核心专题”。
   - **内部 Prompt 示例**：“请为这篇《大语言模型安全报告》写一段来源摘要，并从中提炼出 1-3 个独立的核心专题（例如：大模型对齐技术、提示词注入攻击），输出其研究边界。”
   - **结果**：生成一篇干净的 `source-note` 和多个带有 `manual_review` 标记的 `topic-page` 雏形。

## 场景二：专题深耕与体系维护 (Topic Cultivation)

**典型情境**：过了一段时间，你发现某个专题（比如“AI 提示词工程”）下积累了不少卡片。你想看看系统能不能帮你找出新的洞察，或者查漏补缺。

**执行命令**（针对特定任务调用特定 Workflow）：
```bash
python scripts\personal_kb_steward.py task "探索专题洞察：AI提示词工程" --apply
```

**背后调用的技能与效果**：

5. **`topic-insight-miner` (专题洞察挖掘机)**
   - **它的工作**：扫描专题下的所有关联卡片，找出你可能没注意到的底层冲突或趋势。
   - **内部 Prompt 示例**：“阅读这篇 Topic 及其引用的 8 篇来源卡片。提取出卡片之间互相矛盾的观点，或者推演出一个更高维度的隐藏洞察。”

6. **`knowledge-gap-finder` (知识盲区探测雷达)**
   - **它的工作**：评估专题的成熟度。
   - **内部 Prompt 示例**：“基于目前该 Topic 下记录的所有内容，指出在‘理论支撑’、‘案例验证’或‘反方观点’方面还缺失了哪些维度的信息？”
   - **结果**：生成一份 `gap-report`（盲区报告），指导你接下来的阅读和调研方向。

## 场景三：战时输出与物料组装 (Writing Preparation)

**典型情境**：你需要写一篇关于《地方媒体的 AI 转型路径》的万字长文或做一次深度演讲。你不想从零开始翻找，你希望管家把你过去一年攒下的相关弹药全拿出来。

**执行命令**：
```bash
python scripts\personal_kb_steward.py task "准备写作素材：地方媒体AI转型" --apply
```

**背后调用的技能与效果**：

7. **`writing-evidence-harvester` (写作证据收割机)**
   - **它的工作**：爬梳与该选题相关的所有概念页、案例和日志，提取硬性支撑。
   - **内部 Prompt 示例**：“围绕‘地方媒体AI转型’，从传入的 20 篇相关笔记中，提取所有明确的数据、核心论点（Claim）、以及对应的成功/失败案例。忽略所有空洞的废话。”

8. **`writing-material-pack` (写作弹药包组装)**
   - **它的工作**：将收割来的证据，组装成结构化的写前大纲包。
   - **内部 Prompt 示例**：“请将这些收集到的论点和数据，按照‘背景痛点 - 破局尝试 - 案例支撑 - 未来展望’的逻辑流进行梳理，生成一份包含详细双链引用的写作素材包。”
   - **结果**：在 `material-packs/` 目录下生成一个即取即用的高密度 Markdown 文件。你只需要照着它填肉，文章就写完了。

## 场景四：生态健康与质量拦截 (Health & Maintenance)

**典型情境**：你的系统是活的，必然会产生断链、没有打标签的文件，或是 LLM 提炼出错产生的低质量页面。系统通过安全门禁与健康巡检保证知识库不被污染。

**执行命令**（定期巡检与队列确认）：
```bash
python scripts\personal_kb_steward.py healthcheck
python scripts\personal_kb_steward.py review list
```

**背后调用的技能与效果**：

9. **`kb-lint-healthcheck` (知识库健康体检)**
   - **它的工作**：不使用 LLM，极速扫描全库 Markdown 语法、YAML 元数据。
   - **验证项**：是否存在不存在的 `[[空链接]]`？是否缺少 `status` 字段？
   - **结果**：生成包含 P0 ~ P3 风险等级的 Lint Report，并将严重问题推入待办队列。

10. **`manual-review` (人工安检处)**
    - **它的工作**：拦截机器拿捏不准的操作。
    - **拦截场景示例**：
      - 长文太杂乱，`raw-ingest-router` 分类置信度过低。
      - `topic-research-compile` 自动创建了一个新的 `topic-page`，需要你确认这个 Topic 的命名是否准确。
      - 系统发现大量断链。
    - **使用方式**：使用 `review show <ID>` 查看详情，`review approve <ID>` 放行。

---

## 💡 总结流转：

1. **每天下班**：把碎片和文章丢入 `raw/`，跑一遍 `run --apply`。
2. **月末回顾**：跑一遍 `healthcheck`，清空 `review list`。
3. **遇到输出任务**：直接命令管家：`task "准备写作素材：XX主题"`。
4. **管家会告诉你**：你现在有什么证据，你还缺什么（Gap），并在你的 `index.md` Dashboard 上时刻展示你最活跃的思想进度。
