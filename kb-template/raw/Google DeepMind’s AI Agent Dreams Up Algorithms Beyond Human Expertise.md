# 📑 [Google DeepMind’s AI Agent Dreams Up Algorithms Beyond Human Expertise](https://www.wired.com/story/google-deepminds-ai-agent-dreams-up-algorithms-beyond-human-expertise/?_bhlid=335d3e3d91e87d620efb0dd4b0be463faaeb05d2)



嘿伙计们，最近看到一则DeepMind的新闻，感觉挺炸裂的。他们搞了个叫AlphaEvolve的AI，这玩意儿能自己设计算法，而且效果比人类专家搞了几十年的还牛。

**[AI学会自造算法，人类饭碗要端不住了？]**

**导语：**这可不是小打小闹，它最亮眼的是在矩阵计算里，找到了比沿用了56年的经典“烹饪食谱”——Strassen算法更“省料”的新配方，实实在在提升了计算效率。它还在数据中心调度、芯片设计这些硬骨头领域找到了更优解。

**我们在想：**

*   这事儿最让人细思极恐的是什么？ AlphaEvolve竟然还能优化设计像Gemini这种“智能助手”背后的算法，等于说AI在自己给自己升级“设计大脑”的能力。
*   我们在想：参照DeepMind自己过去几年在算法发现上的节奏——比如2022年的AlphaTensor，再到2024年的Fun Search，现在又来了AlphaEvolve——你会发现AI发现新算法的速度正在明显加快。这就像“自然选择的筛选器”找到了一个加速度旋钮。我们预测，如果这个加速趋势保持下去，**可能在未来3-5年内（基于DeepMind近年的研发迭代周期），AI在特定复杂算法设计上的突破速度将达到人类专家的数倍，让很多传统算法设计领域面临颠覆性挑战。** 这不是科幻，这是正在发生的事实，**数据来源就是DeepMind公开的研究成果迭代速度。**
*   这背后其实藏着一个技术悖论：我们用AI这个“智能助手”来设计更“聪明”的“烹饪食谱”（算法），结果它设计的食谱又反过来让AI自己变得更聪明，形成一个正向飞轮。但问题来了，如果这个飞轮转得太快，我们人类还能跟上理解和控制它设计出来的这些“黑箱”食谱吗？
*   风险在哪里？它能减少计算步骤，听起来是好事，对吧？比如某个计算量巨大的任务，原本需要100步，它可能优化到只用80步，效率提升了20%。但这就像一个更高效的“数字放大镜”，如果它自己带着某种我们不理解的“扭曲视角”（潜在的未知偏见），那它设计出来的所有基于这个视角的算法，可能都会把这种扭曲放大，影响的可是从数据中心到芯片再到我们用的各种AI应用。**这种效率的提升是以我们对底层逻辑的可解释性下降为代价的，这是一个需要警惕的比例失衡。**
